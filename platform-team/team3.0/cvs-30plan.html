The 3.0 plan below is a proto plan and input from the community is very welcome.

Themes for Team 3.0

Improve support for ISVs creating Team plugins

In 2.0 Team plugins could easily integrate into the workbench but contrary to 1.0 we didn't provide many re-useable Team specific views/actions. This meant that a substantial amount of work was required to write a plugin that has the same level of integration as CVS.

  • Sync view re-design and API effort (M3)
  • Improve team API (37705) (M5)
  • Fix the problems with the Team hooks and team-private mechanism (37722)
Proposed:
  • Make a generic repo view
  • Create common team action sets
  • Enhances project set support
  • Add team events.

Work on the 3.0 plan items that affect Team

There are several other items on the Eclipse plan that is of much interest to the Team component. The following are those that we will be participating in:

  • Support logical resources (37723) (M3)
  • User settings (36965)
  • Concurrency and background tasks (36957) (M3)
  • Improve file encoding support (37933)
  • Allow editors to open remote files (37935)
  • Content-type-based editor lookup (37668)

Evolve the CVS plugin

We will continue to fix important bugs and provide key enhancements to the plugin.

  • Integrate into the new sync view
  • Provide annotate command
  • Allows some CVS operations to run in the background
    • synchronizing
    • check outs
    • commits
    • tagging (versions and branches)
  • Remove CVS plugin from platform SDK
    • make a CVS component separate from Team (bugzilla, web pages)
    • make install/update feature for CVS
  • Improve repositories view (proposed)
    • re-work tag management
    • improve configure tags dialog
  • Allow editing of remote files
    • Extend RemoteFileEditor to allow save (commit)
    • Expand repository view UI for opening files for viewing, editing, and external viewing
  • Improve merging and branching workflows
  • Improve initial sharing
    • show files that are being ignored
    • perform an add instead of the import
  • Improve the performance of the CVS decorators (M4)
  • Implement an improved revert mechanism (this will replace the 'copy all right to left' in compare editor)
  • Support shallow commands (e.g. all commands that support -l)
  • Permission support (see bug 20575)
