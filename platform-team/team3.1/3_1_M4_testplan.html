3.1 M4 Team/CVS Test Plan

The goal of this test pass is to test the funtionality of the Team/CVS component that has been modified during this milestone. Click here for a list of all the bugs fixed during this milestone.

Checklist

The following list sumarizes the areas of the Team/CVS component that require testing:

  • Verify the bugs fixed this milestone.
  • File Content Type:
    • The file content type now supports filename, as well as file extension, based matching.
  • Commit wizard:
    • user is now prompted for unknown file content types
    • a preview of the commit is now shown under the comment text pane
  • Tag Filtering
    • Many tag selection dialogs (replace, compare, switch tags) now support tag filtering
  • Branching and Merging
    • branch dialog supports content assist on branch tag text that shows tags from other local projects
    • merge wizard supports content assist on end and start tag
    • merge wizard supports merge into local workspace with end tag only (i.e. start tag not required)
  • Keybindings and top-level CVS menu
    • The main CVS commands are now key bindable.
  • Tag Refresh
    • tag refreshing has been improved in cases where an auto-refresh file does not exist
    • also, improved for cases where the local project maps to a non-root folder in the repository
