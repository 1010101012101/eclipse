Eclipse 3.1 Performance

The performance team will be assisting other teams in their performance work in the following ways:

  1. Adding performance monitoring by default to all non-milestone builds in M6 and M7. This will log any performance issues to the performance.log file in the metadata of your workbench which can be used to enter any problem reports to report these errors.

  2. Analysis of performance issues found in our own testing. We will do performance testing ourselves and log problem reports with the steps to create the performance issue and the resulting metric result which can be used to calculate performance improvements.

  3. Creation of a performance preference page so that the user can turn off features that can affect performance. If these features are off by default we will not add them to this page. We will remove them from the page if they become performant enough to no longer be an issue.

  4. Tracking of existing performance bugs:

    • Outstanding performance bugs organized by team with proirity P1-P3.
    • Completed performance bugs with the improvements listed
