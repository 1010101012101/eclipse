Contributing a port to an Eclipse build
DRAFT

Last updated: June 15, 2005


The Eclipse platform team has limited hardware and platform resources to compile and produce builds of Eclipse on a wide variety of platforms. This document describes the mechanism to provide a build for a opearating system-windowing system-architecture (os-ws-arch) that is not currently built by the platform team. In assuming this role, teams or individuals will be expected to perform the following tasks. 

  • Meet eclipse.org  legal requirements.
  • Contribute patches required for build and run Eclipse on the new platform
  • Run the build and conduct automated tests at a regular interval.
  • Resolve os-ws-arch specific bugs
  • Upload a tested build for inclusion in upcoming Eclipse platform milestones and releases to the eclipse.org infrastructure.
These conditions must be met for new port to be provided on eclipse.org.

The process to include an external build on eclipse.org is a follows:
  1. Open a bug with the Product Platform and Component Releng describing the os-ws-arch drop and type of drop you intend to provide. For instance, will you provide the SDK only?  Or will you provide both the SDK and platorm runtime.  The platform releng team will arrange for PMC approval of support of this new drop. This may also require that a new project is created to provide swt libraries for this platform.
  2. Once approval is granted, use the  Source Build or Source Fetch build scripts to generate a build if you are contributing the SDK only.  If you are interested in contributing other components, please use the instructions to generate a Eclipse components from the command line using Ant scripts in this project and org.eclipse.releng.basebuilder. Incorporate platform specific patches in these scripts and test the resultant build.  A bug describing the patches to ensure that the build runs should be opened with Product Platform and Component Releng.  We will incorporate these patches into our build scripts or ensure that they are included in other teams code base as appropriate.
  3. The cvs tag of  the builder projects (org.eclipse.releng, org.eclipse.releng.eclipsebuilder and org.eclipse.releng.basebuilder) that was used to create the build will be listed on the test results page of the build. Refer to the platform releng build schedule if you need additional information about upcoming builds
  4. Run the automated tests for your platform after you have successfully completed the build.
  5. Request a ftp account on download1.eclipse.org from webmaster@eclipse.org
  6. Copy the build to the staging area for eclipse downloads.

    7. /home/data/httpd/download-staging.priv/eclipse/downloads/<yourorganization>

  7. Once you have pushed your files to the server, notify the releng team that your file available. They will move it to the appropriate directory so it's available for download from eclipse.org.
