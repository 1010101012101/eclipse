3.0 M9 Ant / External Tools Test Plan

Test script for platform Ant support in 3.0 Eclipse SDK, M9 test pass. This test plan focuses on, but is not limited to, features that are new in Eclipse 3.0.

Testers

Committers: Jared Burns, Darin Swanson, Darin Wright

Community: Kevin Barnes, Luc Bourlier, and anyone else

Testing will cover the functional areas in the following list on these platforms/ Ant versions

Linux Luc Jared

Feature Testing

Templates (Kevin Barnes/Luc Bourlier)

  • presented in reasonable contexts when code assisting
  • can add / remove / edit and restore using the preference page

Formatting in the Ant Editor(Darin Swanson / Jared Burns)

  • auto indent strategy works for both newlines and pasting
  • buildfile is formatted per the preferences
  • profile for performance / memory leaks

Code assist in the Ant Editor (Kevin Barnes/Darin Swanson)

  • custom tasks (use the test cases in the tests as a beginning template):
    • boolean attributes
    • reference attributes
    • enumerated attributes
  • macrodefs (use the test cases in the tests as a beginning template):
    • attributes
    • nested elements
    • nested element attributes
    • tasks in the sequential task

External element and Imports (Darin Swanson)

  • outline generated correctly
  • "Open with" support works as expected
  • properties defined correctly (check hover values)
  • console hyperlinking for errors from imported or external entities

External Build files (Kevin Barnes/Luc Bourlier)

  • correct syntax coloring
  • outline generated correctly
  • build using the editor as context or from the editor outline context menu
  • editor updated based on the editor prefs
  • text hovering - for properties, target descriptions and referenced objects such as paths
  • with and without problems

Ant preference page preview viewers (Jared Burns / Darin Swanson)

  • syntax preview
  • template preview
  • formatter preview

Ant launch configuration build tab (Darin Swanson)

  • scope the build
  • turn off build before launching

Ant Editor performance and polish (Darin Swanson)

  • while creating the buildfile
  • opening / closing / saving
  • content assist
  • annotations and problems
  • hover

Ant integration use of variables (Darin Wright / Kevin Barnes)

  • for properties
  • on the classpath
  • environment vars

General Ant builds and Builders (Darin Wright / Darin Swanson)

  • foreground / background builds
  • on older VMs
  • triggering of builders from clean etc
  • headless builds using the AntRunner application entry point
