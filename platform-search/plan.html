Platform Search
Overall 3.0 Plan
The 3.0 plan below is a revised plan for the milestones M7 - M9 and still under developement. Input from the community is very welcome.

Top Items

  • Add support for search participants
  • Support different search result presentation (hierarchical, flat, graphical, ...)
  • Support searching in background 

Planned items

  • Architectural issues 
    • add support for search participants
      • define new extension point
      • change Search UIs to honour participants (search pages and menu items)
    • support to search in the background
  • Regular expression support
    • regular expression support in text search engine
    • option to search for regular expression on File search page
    • get rid of the StringMatcher class
    • Improved replace feature:
      • make it more visible (currently one has to know that this function is available in the Search view's context menu)
      • offer automatic global replace (24008)
    • Search comfort
      • add support for a project scope (16287)
      • change history drop down into Previous and Next buttons like the navigation history
      • (add 'Text Search' to editor context menus (19068))
    • Improve file encoding support (36950)

    Investigations

    • search (for) files which are outside the workspace (29389)
    • search in archives (23341)
    • (allow regular expression in replace string)

    Cross team dependencies

    • Search in the background depends on Platform Core and Platform UI support (36957)
    • Support for regular expressions: JDT Core and JDT UI should also add support for regular expressions
    • Search participants: plug-in which contribute to search should honor the new search participants extension point
