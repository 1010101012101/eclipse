Request For Comment 

Eclipse Platform Core RFC 0002

Importing and Exporting Projects

Summary
The goal is to provide a basic and extensible way of sharing projects when a repository is not required.

By Rodrigo Peretti, OTI
Last Modified: December 04, 2001

Motivation

Sometimes when prototyping some features, developers do not want or do not need to use a repository. But the developer usually wants to have a way of sharing or persisting the projects that contain these features. This sharing can be something like sending an email with the project, moving the project to a new workspace, etc... Moving projects from one workspace to another using copy/cut and paste could become possible through these APIs.
The current APIs available on Resources plug-in loosely expose ways of doing a feature similar do the one being proposed. This RFC proposes more consistent APIs to allow this kind of sharing. One of the goals is to start with very few metadata information and add more as more use cases come up.

Proposed API

The current proposal is already in an API format. The API is going to be added to IWorkspace and consists of:

Constants:

  • METADATA_ALL
  • METADATA_PROJECT_DESCRIPTION
Methods:
  • availableProjects
  • exportProjects
  • importProjects

RFC Status

This proposal is not currently being voted on. However, you should:
  • Provide all comments/feedback to the mailing list.
  • Clearly indicate if you are fundamentally opposed to the proposal and provide reasons.
If you have serious concerns don't wait until there is a vote - speak up now!
