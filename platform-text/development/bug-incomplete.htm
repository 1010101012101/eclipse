Incomplete Bug Report

Bug reports and feature requests which do not provide enough information to be tackled are either resolved to INVALID or WORKSFORME if we think it's not a bug. It is the reporters task to first fill in all needed information which is necessary and then reopen the report. Missing important information includes:
  • build id - the Build id can be found in the About dialog (Help > About Eclipse Platform)
  • are other plug-ins installed? If so, which ones?
  • VM information - if not sure start eclipse using -vmargs -showversion
  • are there related entries in the .log file? - the log file is <workspace location>/.metadata/.log
    • if no, write it in the bug report
    • if yes, attach all related entries (it's better to attach the whole .log than miss related entries)
  • attach VM dump in case of VM crashes - normally the dumps can be found in the Eclipse install location
  • attach stack dumps in case of deadlocks (how to report a deadlock)
  • attach stack dumps in case things that take (too) long - (how to get a stack trace)
  • attach preferences - if you think it depends on some preferences (e.g. worked before)
  • attach formatter profile - if the report is about badly formatted code
  • specify the concrete editor for editor bugs, e.g. "copy broken in Text editor" instead of "copy does not work in editor"
  • provide the keyboard layout and Locale if the bug is about key bindings not working or weird characters appearing on typing
  • try to provide steps to reproduce the problem and attach test resources (if possible)
  • tell us if you write code against J2SE 5.0 i.e. compiler compliance level set to 1.5?
