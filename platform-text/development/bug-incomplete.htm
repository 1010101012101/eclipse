Incomplete Bug Report


Bug reports and feature requests which do not provide enough information to be tackled are either resolved to REMIND or WORKSFORME if we think it's not a bug. It is the reporters task to first fill in all needed information which is necessary and then reopen the report. Missing items include:
  • build id - the build id can be found in the About dialog (Help > About Eclipse Platform)
  • VM information - if not sure start eclipse using -vmargs -showversion
  • are there related entries in the .log file? - the log file is <workspace location>/.metadata/.log
    • if no, write it in the bug report
    • if yes, attach all related entries (it's better attach the whole .log than miss related entries)
  • attached preferences - if you think it depends on some preferences (e.g. worked before)
  • attached formatter profile - if the report is about badly formatted code
  • specify the concrete editor for editor bugs, e.g. "copy broken in Text editor" instead of "copy does not work in editor"
  • try to provide steps to reproduce the problem and attach test resources (if possible)
